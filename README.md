# Теңге Симулятор 🇰🇿

Вирусная игра-угадайка курса USD/KZT. Тебе дают **1 000 000 ₸** и случайный кусок
реальной истории курса (даты скрыты). 10 раундов ставишь «Доллар ⬆» или «Доллар ⬇»
на следующий торговый день. В конце — вердикт от «Сорос из Астаны» 🦅 до
«Февраль 2014 в душе» 💀, раскрытие эпохи, в которой ты торговал, и шер-карточка.

Не является инвестиционной рекомендацией 😉

## Запуск

```bash
npm install
npm run dev        # http://localhost:3000
npm test           # vitest: движок + датасет (23 теста)
npm run build      # production-сборка
```

## Как это устроено

- **`lib/engine.ts`** — чистая игровая логика: детерминированный PRNG (mulberry32),
  выбор сегмента из 70 торговых дней (60 видимых + 10 игровых), математика выплат
  (выигрыш = ставка × max(5%, |движение курса| × 20)), банкротство, титулы, перцентили.
  Полностью покрыта тестами (`lib/engine.test.ts`, `lib/dataset.test.ts`).
- **Seed в URL** (`?s=…`) — одна и та же ссылка всегда даёт один и тот же кусок
  истории: «сыграй мой сценарий».
- **~30% игр** нарочно попадают игровыми днями в горячие эпохи: февраль 2014,
  август 2015, март 2020, февраль–март 2022.
- **`data/usdkzt.json`** — 3392 реальных официальных курса НБ РК (будни
  2013-01-01 → 2025-12-31), закоммичен в репозиторий: в рантайме ничего не качается.
- **`/api/og`** — динамическая OG-картинка (`next/og`) для шеринга результата.
- Состояние игры — `useReducer` на клиенте; рекорд и счётчик игр — localStorage.

## Обновить датасет

```bash
npm run fetch-rates
```

Скрипт `scripts/fetch-rates.ts` сначала пробует stooq.com (один CSV-запрос), при
неудаче собирает официальные курсы с RSS Нацбанка Казахстана
(`nationalbank.kz/rss/get_rates.cfm`) пулом по будним дням — это займёт пару минут.
Скрипт отказывается писать файл, если получил меньше 2500 точек, и печатает
sanity-check девальвации августа 2015 (188.38 → 255.26).
